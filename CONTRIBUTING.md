# Contributing to TerraLens

Thank you for your interest in contributing to TerraLens! We welcome contributions from the community to help make this geometric optimization engine even better.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please open an issue on GitHub. Include:
- A clear description of the bug.
- Steps to reproduce the issue.
- Details about your environment (OS, Python version, etc.).

### 2. Suggesting Enhancements
We are always looking for ways to improve TerraLens. If you have an idea:
- Check if a similar suggestion has already been made.
- Open an issue describing the proposed enhancement and its benefits.

### 3. Code Contributions
- **Fork the repository** and create your branch from `main`.
- **Install dependencies** as specified in the README.
- **Write clean code** following the existing style.
- **Add tests** for any new functionality.
- **Ensure all tests pass** before submitting.
- **Submit a Pull Request** with a clear description of your changes.

## Coding Standards
- Use consistent indentation (4 spaces for Python, standard for C++).
- Include docstrings and comments for complex logic.
- Follow PEP 8 for Python code.
